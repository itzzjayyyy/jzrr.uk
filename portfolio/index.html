Tower Defense Game
           
I have created many games in Roblox Studio, and one of the most ambitious is a tower defense game. Although the project is still in development, it has already given me valuable experience in designing complex systems and ensuring smooth gameplay. A key part of this type of game is the placement system, which lets players place towers strategically on the map. Implementing this required careful thought about grid alignment, collision detection, and user interface feedback to make placements feel intuitive and responsive. To make the game secure and reliable, I focused heavily on client-server communication. In Roblox, this means using RemoteEvents to send placement requests from the client to the server, where all critical checks happen. By validating tower positions and player resources on the server side, I could prevent exploits like free placements or bypassing cooldowns. This approach keeps the game fair and competitive, even in multiplayer settings. Beyond placement mechanics, I’ve also worked on wave spawning systems, enemy pathfinding, and damage calculation logic. Each of these systems is essential to a tower defense game. They required modular scripting and event-driven design so they could be easily extended as the game grows. For instance, adding new tower types or enemy behaviors can happen without rewriting the entire codebase. This is thanks to how I structured the scripts. This project has improved my skills in secure scripting, modular design, and gameplay balancing. It has also taught me how to anticipate edge cases and build systems that scale. It shows my ability to combine creativity with technical skill, producing games that are not only fun to play but also strong in their performance.

Discord Bot
I have developed several Discord bots, each with its own unique features and challenges. One of my most notable projects was an invite tracker bot. This bot was meant to monitor server growth and accurately link new members to the right inviter. Building it required a solid understanding of JavaScript and the Discord.js library. I also needed to manage complex math and positioning logic to ensure every invite was tracked correctly, even in tricky situations like deleted or expired links. To make it work well, I included strong error handling and server-side validation checks. This made the bot reliable during heavy use and protected it from exploits. The system was modular, which let me easily add new features like leaderboards, persistent storage, and dynamic reporting. This project also gave me experience in data visualization. I created a custom welcome card generator using Node-Canvas. The card displays the user’s avatar, server information, and stylized text, providing a polished and engaging experience for new members. Beyond this bot, I've built moderation tools, notification systems, and fun, context-aware response features. Each project has helped me improve my skills in event-driven programming, API integration, and modular design. I also learned how important it is to create bots that are both technically sound and enjoyable for communities.
